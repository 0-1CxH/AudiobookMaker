# ABM - 详细UX设计

基于UX.md和源码分析的详细页面设计文档。

```
项目工作流: 1.项目选择 → 2.角色提取 → 3.对话分配 → 4.语音设计 → 5.音频生成
```

### API端点使用
```python
# 1. 获取所有项目列表
GET /api/projects

# 2. 创建新项目
POST /api/projects
{
  "name": "项目名称",
  "raw_text": "原始文本内容",
  "project_setting": {
    "split_format": "line",
    "quote_format": "auto",
    "context_window": 10,
    "voice_lib_folder_path": "./workspace/voice_lib",
    "design_model_path": "./workspace/design_model",
    "clone_model_path": "./workspace/clone_model"
  }
}

# 3. 加载项目
GET /api/projects/<project_id>

# 4. 自动文本处理（创建后自动执行）
POST /api/projects/<project_id>/text/process-text
```

### 交互细节
1. **项目列表**: 显示所有workspace/projects目录下的项目，按修改时间排序
2. **创建表单**:
   - 文本区域支持粘贴或上传.txt/.md文件
   - 项目设置继承默认配置，可调整
   - 点击"创建并开始处理"后，自动执行文本分割和引语检测
3. **加载流程**: 点击"加载"按钮后，调用`Project.load()`方法，进入页面2

### API端点使用
```python
# 1. 获取对话分配状态
GET /api/projects/<project_id>/dialogues
# 返回: {segment_index: speaker_name, ...}

# 2. 自动分配对话
POST /api/projects/<project_id>/dialogues/allocate
# 对应: project.generate_quote_allocation()

# 3. 更新单个对话分配
PUT /api/projects/<project_id>/dialogues/<segment_index>
{
  "speaker": "角色名"
}
# 对应: project.set_quote_allocation(segment_index, speaker)

# 4. 批量更新
POST /api/projects/<project_id>/dialogues/batch
[
  {"segment_index": 0, "speaker": "刘备"},
  {"segment_index": 1, "speaker": "关羽"}
]

# 5. 删除分配（重置为QUOTE_TAG）
DELETE /api/projects/<project_id>/dialogues/<segment_index>
# 对应: project.remove_quote_allocation(segment_index)
```

### 交互细节
1. **文本颜色编码**:
   - 灰色: default_tag, placeholder_tag（不可分配）
   - 透明背景黑色文字: 未分配的引语（可点击）
   - 彩色背景: 已分配的引语，背景色对应角色颜色
2. **点击分配**: 点击未分配引语弹出气泡选择角色，选择后立即生效
3. **角色颜色**: 每个角色分配唯一颜色，用于可视化
4. **AI对话分配**: 调用LLM根据上下文自动分配所有引语
5. **验证**: 进入下一步前检查是否所有引语都已分配

### API端点使用
```python
# 1. 开始生成音频（异步任务）
POST /api/projects/<project_id>/audio/generate
# 对应: project.generate_text_to_audio_segment()
# 返回: {"task_id": "task_123", "message": "音频生成任务已开始"}

# 2. 获取生成状态
GET /api/projects/<project_id>/audio/status/<task_id>
# 返回: {"status": "running/completed/failed", "progress": 85, "completed": 34, "total": 40}

# 3. 获取音频片段列表
GET /api/projects/<project_id>/audio/segments
# 返回: [{index: 0, content: "...", speaker: "...", status: "generated/not_generated", audio_path: "...", ...}]

# 4. 重新生成单个音频片段
POST /api/projects/<project_id>/audio/segments/<segment_index>/regenerate
# 对应: 删除原音频后重新调用generate_text_to_audio_segment逻辑

# 5. 获取音频生成进度
GET /api/projects/<project_id>/audio/progress
# 返回: {"generated": 34, "total": 40, "percentage": 85}

# 6. 渲染最终音频
POST /api/projects/<project_id>/output/render
# 对应: project.render_audio()

# 7. 下载最终音频
GET /api/projects/<project_id>/output/download
# 返回音频文件流

# 8. 获取输出状态
GET /api/projects/<project_id>/output/status
# 返回: {"final_audio_exists": true, "file_size": 44503200, "duration": 2732.5}
```

### 交互细节
1. **进度显示**: 实时显示生成进度百分比和数量
2. **状态颜色**:
   - 黑色文字: 已生成的片段
   - 灰色文字: 未生成的片段
3. **AI音频生成**: 批量生成所有未生成的音频片段
4. **单个操作**: 可单独生成、重新生成或播放每个片段
5. **片段详情**: 点击片段显示详细信息对话框
6. **最终渲染**: 所有片段生成完成后，可渲染最终音频文件
7. **下载**: 渲染完成后提供下载链接

## 前端-后端数据流

### 页面1 → 页面2
1. 用户输入文本 → POST /api/projects
2. 后端创建Project实例，自动处理文本
3. 返回项目ID，前端跳转到页面2
4. GET /api/projects/<id>/text/segments 显示文本
5. GET /api/projects/<id>/characters 显示角色（初始只有"默认"）

### 页面2 ↔ 页面3
1. 页面2添加/编辑角色 → POST/PUT /api/projects/<id>/characters
2. 进入页面3时，GET /api/projects/<id>/dialogues 获取分配状态
3. 手动分配 → PUT /api/projects/<id>/dialogues/<segment_index>
4. AI分配 → POST /api/projects/<id>/dialogues/allocate

### 页面3 ↔ 页面4
1. 进入页面4时，GET /api/projects/<id>/voice/designs
2. AI语音设计 → POST /api/projects/<id>/voice/generate-designs
3. 生成参考音频 → POST /api/projects/<id>/voice/generate-reference-audio

### 页面4 ↔ 页面5
1. 进入页面5时，GET /api/projects/<id>/audio/progress
2. 生成音频 → POST /api/projects/<id>/audio/generate (异步)
3. 轮询状态 → GET /api/projects/<id>/audio/status/<task_id>
4. 渲染最终 → POST /api/projects/<id>/output/render

## 错误处理提示

### 常见错误场景
1. **LLM API错误**: "AI服务暂时不可用，请检查网络和API密钥"
2. **TTS模型缺失**: "未找到TTS模型文件，请在项目设置中配置正确路径"
3. **磁盘空间不足**: "磁盘空间不足，无法保存音频文件"
4. **项目加载失败**: "项目文件损坏或格式不正确"
5. **音频生成失败**: "音频生成失败，请检查TTS模型和语音设计"

### 用户提示位置
- 按钮下方小字提示
- 页面顶部横幅警告
- 弹窗确认对话框
- 进度条错误状态
